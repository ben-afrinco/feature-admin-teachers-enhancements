## Core Identity & Authentication

### `user` Table
Stores primary authentication and identity details for all system users (Students, Teachers, Admins).

| Column | Type | Description |
| :--- | :--- | :--- |
| `user_id` | BIGINT (PK, AI) | Unique identifier for the user. |
| `full_name` | VARCHAR(100) | The user's displayed name. |
| `email` | VARCHAR(100) | Unique email address (used for login). |
| `password` | VARCHAR(255) | Hashed password. |
| `role` | VARCHAR(20) | User role: `student`, `teacher`, or `admin`. |
| `created_at` | TIMESTAMP | Record creation time. |
| `updated_at` | TIMESTAMP | Record update time. |

## User Roles & Entities

### `admin` Table
Link table for administrator-specific metadata.
- **Relationships**: `user_id` (FK -> `user.user_id`)

### `teachers` Table
Link table for teacher-specific metadata.
- **Relationships**: `user_id` (FK -> `user.user_id`)

### `student` Table
Main entity for test-takers.
- **Relationships**: 
    - `user_id` (FK -> `user.user_id`)
    - `class_id` (FK -> `classes.class_id`)
- **Columns**:
    - `student_id` (PK)
    - `level`: Initial or current English proficiency level.

## Tests & Questions

### `test` Table
Defines the assessment modules.

| Column | Type | Description |
| :--- | :--- | :--- |
| `test_id` | BIGINT (PK) | Unique test identifier. |
| `test_name` | VARCHAR(255) | Title of the test. |
| `level` | VARCHAR(50) | Target proficiency (Beginner, Intermediate, etc.). |
| `skill` | VARCHAR(50) | `reading`, `listening`, `writing`, or `speaking`. |
| `content` | LONGTEXT | For Reading: The story/text. For Listening: The path to the Audio file. |

### `questions` Table
The specific tasks within a test.
- **Relationships**: `test_id` (FK -> `test.test_id`)
- **Columns**:
    - `question_text`: The prompt.
    - `question_type`: `mcq`, `text`, or `voice`.
    - `difficulty_level`: `easy`, `medium`, `hard`.

### `options` Table
Multiple-choice options for MCQ questions.
- **Relationships**: `question_id` (FK -> `questions.question_id`)
- **Columns**: `optione_text`, `is_correct` (boolean).

## Student Responses & AI Evaluation

### `dent_answer` Table
Stores raw student submissions.
- **Relationships**:
    - `student_id` (FK)
    - `question_id` (FK)
    - `option_id`: (FK, nullable) Links to the selected option for MCQs.
- **Column**: `answer_text` (For writing/speaking transcribed text).

### `result` Table
Final scores per test attempt.
- **Relationships**: `user_id` (FK), `test_id` (FK).
- **Column**: `final_score` (Percentage or point-based).

### `ai_evaluations` Table
Deep feedback generated by Gemini/AI models.
- **Relationships**: `answer_id` (FK -> `dent_answer.answer_id`)
- **Columns**: `ai_score`, `ai_feedback` (Longtext).

## Class Management

### `classes` Table
Logical grouping of students under a teacher.
- **Relationships**: `teacher_id` (FK -> `teachers.teacher_id`)
- **Columns**: `classes_name`, `level`, `description`.

### `class_student` Table
Pivot table for Many-to-Many relationship between Classes and Students.
- **Foreign Keys**: `class_id`, `student_id`.
