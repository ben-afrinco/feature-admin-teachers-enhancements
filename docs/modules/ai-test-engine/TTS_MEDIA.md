# TTS and Media Handling

LingoPulse provides a rich auditory experience for the Listening skill by dynamically converting AI-generated scripts into speech.

## 1. TTS Provider: Google Translate API (GTX)

The system uses an unauthenticated proxy to Google's Text-to-Speech engine.

- **Endpoint**: `https://translate.google.com/translate_tts`
- **Parameters**:
    - `ie=UTF-8`: Input encoding.
    - `client=gtx`: Identifies the client as a browser-like request.
    - `tl=en-US`: Target language (American English).
    - `q=...`: The URL-encoded text script.

## 2. Audio Processing Flow

1. **Text Preparation**:
    - The `script` generated by the AI (Llama 3.3) is stripped of any JSON artifacts.
    - The text is shortened to 500 characters using `substr()` to prevent URL length issues and Google's free-tier payload limits.
2. **Download & Persistence**:
    - `file_get_contents()` is used to fetch the raw MP3 binary data from Google.
    - `public_path('storage/audio/')` is checked, and the directory is created if missing (`0755` permissions).
    - Files are saved with a unique name prefixing `dynamic_listening_` followed by a `uniqid()`.
3. **Database Linking**:
    - The path string `storage/audio/filename.mp3` is saved in the `content` column of the `test` record.

## 3. Frontend Implementation

### HTML5 Audio Integration
In the `dynamicListening.blade.php` view, the audio is delivered via the standard `<audio>` tag:

```html
<audio controls controlsList="nodownload">
  <source src="{{ asset($test->content) }}" type="audio/mpeg">
</audio>
```

- **`controlsList="nodownload"`**: Helps prevent casual downloading of the test assets.
- **`asset()` helper**: Ensures the correct URL is generated relative to the public storage link.

## 4. Limitations & Maintenance

- **Rate Limiting**: Since the TTS uses a free unauthenticated endpoint, high volume might trigger 429 errors from Google. The system handles this by skipping the record creation if `file_get_contents` fails.
- **Storage Cleanup**: Currently, files are saved permanently to build the question bank. A future maintenance task may include a command to delete very old or unused media if disk space becomes a concern.
