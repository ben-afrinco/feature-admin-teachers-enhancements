# Project Glossary

Key terms and concepts used within the LingoPulse ecosystem.

| Term | Definition |
|---|---|
| **AI Assessment** | The process of using LLMs (Gemini, Llama) to grade qualitative student work. |
| **Classes** | Logical groupings of students assigned to a specific teacher and level. |
| **Dynamic Test** | A test generated on-the-fly by AI that is unique to the current student. |
| **Gemini** | Google's AI model used for writing and speaking evaluation. |
| **Groq** | A high-performance inference engine used to generate reading tests. |
| **Jitsi** | The open-source video conferencing integration used for classes. |
| **Level** | Proficiency ranking (e.g., A1, B2, Beginner, Intermediate). |
| **Notifiable** | A Laravel trait allowing models (Users) to receive system alerts. |
| **Pivot Table** | A database table used to map many-to-many relationships (e.g., `class_student`). |
| **Private Disk** | Storage partition not accessible by public URLs, used for secure files. |
| **Result Detail** | The comprehensive view of a student's performance across all skills. |
| **Skill Radar** | A spider chart visualization of proficiency in Reading, Listening, etc. |
| **Submission Version** | An iteration of a student's work for an assignment. |
| **Teacher Dashboard** | The interface where teachers manage classes and grading. |
